# 🚀 راهنمای اجرای همزمان فرانت‌اند و بک‌اند

این پروژه (فرانت‌اند) و بک‌اند `spring-banking-platform` **جدا** اجرا می‌شوند و با هم کار می‌کنند.

## گام ۱ — بک‌اند (Spring Boot)
در پوشه‌ی بک‌اند:
```bash
# با Docker (پیشنهادی)
docker compose up -d
```
یا هر سرویس را جدا با Maven اجرا کنید. فقط مطمئن شوید **api-gateway روی پورت 8090** بالاست.

> اگر هنوز بک‌اند را اجرا نکرده‌اید، نگران نباشید: فرانت یک **«حالت دمو»** دارد که با داده‌ی نمونه کار می‌کند.

## گام ۲ — فرانت‌اند (همین پوشه)
در WebStorm پوشه را باز کنید و در ترمینال:
```bash
npm install
npm run dev
```
سپس مرورگر را روی **http://localhost:5173** باز کنید.

## ورود
- **حالت دمو** (سوییچ در صفحه‌ی ورود): بدون بک‌اند، با کاربران `customer` / `manager` / `admin`.
- **اتصال واقعی**: حالت دمو را خاموش کنید؛ همان کاربران در بک‌اند هم وجود دارند:
  - `customer` / `customer123` → پنل کاربر
  - `manager` / `manager123` → پنل مدیر
  - `admin` / `admin123` → پنل ادمین

## نکته‌ی مهم درباره‌ی پروکسی
نیازی به تنظیم CORS نیست: Vite همه‌ی `/api/*` را به `http://localhost:8090` پروکسی می‌کند
(در فایل `vite.config.ts`). اگر آدرس بک‌اند فرق دارد، فایل `.env` بسازید:
```
VITE_API_TARGET=http://localhost:8090
```
